Cookin’ Up a GitHub Pages Site

Welcome to the easey-peasy, super-simple way to set up a website with GitHub Pages!

You don't need to know anything about git or GitHub or cooking!

As long as you’ve got a few basic HTML tags under your belt, you’ll be cookin’ up your own fully-live website in minutes. For free. Like all the best things are.

Ingredients

  • An Internet connection
  • A modern web browser
  • A GitHub account
  • A doctype declaration
  • Some HTML elements
  • And a handful of HTML attributes

Instructions

  1. Make sure you’re connected to the Internet.
  2. Fire up your web browser (everyone uses Chrome, but Firefox is better - yep, I’m one of those people)
  3. Go to www.github.com and log in with your GitHub account details.
  4. On your GitHub home page, click the green New button to create a new repository (just a posh word for folder).
  5. You now need to give your repository a name. Naming things is usually one of the hardest things to do in programming, but this time it’s really easy. You simply take your GitHub username and add .github.io after it. So if your GitHub username is sir-quincy-of-larson, then your repository should be called sir-quincy-of-larson.github.io
  6. You don’t need to change anything else on this page, so just go ahead and click the green Create repository button. 7. On the next page look for the section that starts with “Quick setup …” and click the blue “creating a new file” link to open up a very basic code editor. 8.
