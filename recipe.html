Cookin’ Up a GitHub Pages Site

Welcome to the easey-peasy, super-simple way to set up a website with GitHub Pages!

You don't need to know anything about git or GitHub or cooking!

As long as you’ve got a few basic HTML tags under your belt, you’ll be cookin’ up your own fully-live website in minutes. For free. Like all the best things are.

Ingredients

  • An Internet connection
  • A modern web browser
  • A GitHub account
  • A doctype declaration
  • Some HTML elements
  • And a handful of HTML attributes

Instructions

  1. Make sure you’re connected to the Internet.
  2. Fire up your web browser (everyone uses Chrome, but Firefox is better - yep, I’m one of those people).
  3. Go to www.github.com and log in with your GitHub account details.
  4. On your GitHub home page, click the green New button to create a new repository (just a fancy word for folder).
  5. You now need to give your repository a name. Naming things is usually one of the hardest things to do in programming, but this time it’s really easy. You simply take your GitHub username and add .github.io to the end of it. So if your GitHub username is sir-quincy-of-larson, then your repository should be called sir-quincy-of-larson.github.io
  6. You don’t need to change anything else on this page, so just go ahead and click the green Create repository button.
  7. On the next page look for the section that starts with Quick setup… and click the blue creating a new file link to open up a very basic code editor.
  8. For the Name your file… field, simply type index.html
  9. Now you’re all ready to start typing your HTML code!
  10. When you’ve finished writing your HTML, click the green Commit changes… button (just a fancy way of saying save).
  11. A Commit changes pop-up window will appear. Here you can add some more info about the changes you’ve made, but for now you can use the default commit message, so just click the Commit changes button.
  12. Stare at your computer screen for around 30 seconds or so. Feel free to chant a mantra or perform a ceremonial dance if you wish. Then hit your browser’s Reload button, and if your mantra or dance was well-received by the gods of GitHub, you should see a new section in the right-hand column called Deployments, along with a green tick next to a github-pages entry. (If it says queued, just wait a few more seconds, and try reloading the page again).
  13. Click the Deployments heading to open your Deployments page, then under All deployments, look for the github-pages card and you’ll see a blue link to the website you’ve created. Click on that link, go “Whoah!!!”, and bask in your warm levelled-up glow.
